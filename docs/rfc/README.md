# RFC Guidance

This directory now has two classes of documents:

- **Authority for the headless-runtime refactor**
  - `../architecture/README.md`
- **Historical implementation RFCs**
  - everything else unless explicitly stated otherwise

For architecture decisions in this worktree:

1. Read `../architecture/README.md` first
2. Use older RFCs only to understand legacy code and migration constraints
3. If an older RFC conflicts with the architecture docs, follow the architecture docs

Older RFCs are valuable implementation history, but they are not design authority for the
headless runtime target architecture.
